# Promplet 仕様書 v0.1

## 1. アプリ概要

**Promplet** は、Windowsデスクトップ上に小さく常駐する、定型文・プロンプト入力補助アプリ。

主な用途は、ChatGPT、Claude、Claude CLI、Slack、メール、ターミナルなどに対して、あらかじめ登録した文章をボタンクリックで素早く貼り付けること。

## 2. コンセプト

```text
常に手前に浮かぶ、小型のプロンプト／定型文ボタンバー
```

特徴は以下。

* 小型ウィンドウ
* 常に最前面表示
* 半透明表示
* 好きな位置にドラッグ移動
* クリックしても貼り付け先のフォーカスを奪わない
* ボタンを押すだけで定型文を貼り付け
* グループごとにボタン群を切り替え
* 常駐型アプリとして動作

## 3. 想定環境

* OS：Windows
* 開発言語：C#
* UI：WPF推奨
* ランタイム：.NET 8 以降
* 配布形態：まずはローカル実行アプリ
* データ保存：JSONファイル

## 4. 基本構成

Promplet は、大きく2つの画面を持つ。

### 4.1 メインパレット

普段使う小型ウィンドウ。

役割：

* 定型文ボタンを表示
* グループ切り替え
* ボタンクリックで貼り付け
* 常に最前面
* 半透明
* 非アクティブウィンドウとして動作

### 4.2 ボタンマネージャー

定型文やグループを編集する通常ウィンドウ。

役割：

* グループ追加・編集・削除
* ボタン追加・編集・削除
* 並び替え
* 本文編集
* 貼り付け設定
* 保存・読み込み

メインパレットは特殊な非アクティブウィンドウにするが、ボタンマネージャーは通常のアクティブウィンドウとして扱う。

## 5. メインパレット仕様

### 5.1 表示仕様

* 常に最前面に表示
* 高さ固定
* 幅は可変
* 半透明表示
* 枠なし、または小さなツールウィンドウ風
* ドラッグで移動可能
* タスクバーには表示しない
* Alt + Tab には出さない、または出にくくする
* 閉じる操作では終了せず非表示にする

### 5.2 ボタン表示

* 表示できるボタン数は **1〜10個**
* 選択中グループのボタンのみ表示
* ボタンは縦並びを基本とする
* ボタンはウィンドウ幅いっぱいに表示し、横幅変更に追随する
* ボタンにはプロンプト名のみを主ラベルとして表示
* ボタン本文は薄いプレビューとして最大3行程度まで表示し、はみ出す部分は見切る

### 5.3 グループ切り替え

グループごとにボタン群を管理する。

例：

* AIチャット
* AI CLI
* Slack
* メール
* Terminal
* 編集
* 開発

ComboBoxは非アクティブウィンドウと相性が悪い可能性があるため、初期実装では以下の方式がよい。

```text
[AI] [CLI] [Slack] [Mail] [Term]
```

または、

```text
◀ AIチャット ▶
```

### 5.4 フォーカス非取得

メインパレットは、クリックしても貼り付け先のフォーカスを奪わない。

使用する主なWin32 API / 定数：

* `WS_EX_NOACTIVATE`
* `WM_MOUSEACTIVATE`
* `MA_NOACTIVATE`
* `SetWindowPos`
* `HWND_TOPMOST`
* `SWP_NOACTIVATE`
* `WS_EX_TOOLWINDOW`

これにより、ChatGPTやClaudeなどのWebアプリ入力欄、あるいはターミナルの入力位置を維持したまま貼り付ける。

## 6. 貼り付け仕様

### 6.1 基本方式

貼り付けは以下の方式で行う。

```text
1. 現在のクリップボードを必要に応じて退避
2. 定型文をクリップボードへ設定
3. 貼り付け先に Ctrl + V などを送信
4. 必要に応じてクリップボードを元に戻す
```

キー送信は `SendKeys` よりも、将来的には `SendInput` を推奨。

### 6.2 貼り付け方式

ボタン単位、またはグループ単位で貼り付け方式を選べるようにする。

候補：

* `Ctrl + V`
* `Shift + Insert`
* `Ctrl + Shift + V`
* コピーのみ
* 確認してから貼り付け

初期実装では以下で十分。

* 左クリック：即貼り付け
* 右クリック：コピーのみ

### 6.3 改行処理

ボタンごと、またはグループごとに改行処理を設定できる。

候補：

* そのまま貼り付け
* 末尾改行を削除
* 末尾に改行を追加
* 前後の空白を削除
* 改行をスペースに変換

AIチャットやClaude CLI向けには、**そのまま貼り付け** が重要。

### 6.4 クリップボード復元

貼り付け方式ではクリップボードを一時的に上書きするため、復元機能があると望ましい。

初期実装では、

* テキストクリップボードのみ復元
* 復元ON/OFF設定

でよい。

画像・ファイルなど複雑なクリップボード形式の完全復元は後回し。

## 7. ターミナル対応

Promplet はターミナルへの入力も想定する。

対象例：

* Windows Terminal
* PowerShell
* cmd
* WSL
* Git Bash
* Claude CLI
* Codex CLI
* Gemini CLI
* Python REPL
* Node.js REPL
* DuckDB CLI

### 7.1 AI CLI向け

Claude CLIなどの対話型AI CLIでは、複数行プロンプトを貼り付ける用途があるため、改行あり貼り付けを許可する。

### 7.2 シェルコマンド向け

危険なターミナルコマンドについては、アプリ側で最低限の制限や警告を設ける。

ただし、最終的に入力・実行するのはユーザーであり、過剰に制限しすぎない。

候補：

* 危険ワードを含む場合に警告
* 特定グループでは危険ワード登録不可
* 危険ボタンは確認必須
* Terminalグループでは末尾改行削除を既定値にする

危険ワード例：

```text
rm -rf
del /s
format
shutdown
reboot
mkfs
dd if=
chmod -R 777
chown -R
curl | sh
wget | sh
```

## 8. ボタンマネージャー仕様

### 8.1 グループ管理

以下を可能にする。

* グループ追加
* グループ名変更
* グループ削除
* グループ並び替え
* グループごとの既定設定

グループごとの既定設定例：

* 貼り付け方式
* 改行処理
* クリップボード復元
* 危険ワードチェック
* 表示ボタン数
* パレット幅

### 8.2 ボタン管理

各ボタンは以下の情報を持つ。

* ラベル
* 本文
* 所属グループ
* 表示順
* 有効／無効
* 貼り付け方式
* 改行処理
* クリック時動作
* 危険ワードチェック対象
* 確認ダイアログ有無

### 8.3 編集UI

ブックマークマネージャーのような操作感を目指す。

画面イメージ：

```text
左：グループ一覧
右：選択中グループのボタン一覧
下：選択中ボタンの本文編集
```

必要な操作：

* 追加
* 編集
* 削除
* 複製
* 上へ移動
* 下へ移動

## 10. 常駐仕様

Promplet はタスクトレイ常駐型とする。

### 10.1 トレイ機能

トレイアイコンから以下を操作できる。

* パレットを表示
* パレットを非表示
* ボタンマネージャーを開く
* 設定を開く
* JSON再読み込み
* 終了

### 10.2 閉じるボタン

メインパレットの閉じる操作は終了ではなく非表示にする。

本当に終了する場合は、トレイメニューから終了する。

## 11. ホットキー仕様

グローバルホットキーを設定可能にする。

初期候補：

```text
Ctrl + Alt + Space
```

用途：

* パレット表示／非表示
* 次のグループへ切り替え
* 前のグループへ切り替え
* ボタンマネージャーを開く

初期実装では、表示／非表示だけでよい。

## 12. 設定保存

保存したい設定：

* パレット位置
* パレット幅
* 透明度
* 最後に選択していたグループ
* クリップボード復元ON/OFF
* ホットキー
* 貼り付け方式の既定値
* 改行処理の既定値

複数モニター環境で画面外に出た場合は、次回起動時に画面内へ戻す。

## 13. 非機能要件

### 13.1 軽量性

* 常駐しても重くならない
* 起動が速い
* メモリ使用量を抑える

### 13.2 安定性

* 貼り付け失敗時にクラッシュしない
* JSON破損時はバックアップを作って初期化
* クリップボード取得失敗時も処理を継続

### 13.3 安全性

* 危険コマンドは登録時または貼り付け時に警告可能
* 貼り付け内容は外部送信しない
* ローカル完結を基本とする

## 14. 開発フェーズ

## Phase 1：最小プロトタイプ

目的：
まず「非アクティブ常時最前面パレットからボタンで貼り付け」が動くことを確認する。

実装範囲：

* WPFアプリ作成
* 小型ウィンドウ表示
* 常に最前面
* 半透明
* ドラッグ移動
* フォーカスを奪わない
* ボタンを数個固定表示
* ボタンクリックでクリップボード貼り付け
* `Ctrl + V` 送信

この段階では、JSON管理やマネージャーは不要。

## Phase 2：JSON定義対応

目的：
ボタンやグループをコードではなくJSONから読み込めるようにする。

実装範囲：

* JSONファイル読み込み
* グループ管理
* 選択中グループのボタンのみ表示
* ボタン1〜10個対応
* ウィンドウ位置保存
* 最後のグループ保存

## Phase 3：常駐化

目的：
実用アプリとして常駐できるようにする。

実装範囲：

* タスクトレイ常駐
* 表示／非表示
* 終了メニュー
* 設定再読み込み
* グローバルホットキー
* 閉じる操作で非表示

## Phase 4：貼り付け設定強化

目的：
Webアプリ、AI CLI、ターミナルなど複数用途に対応する。

実装範囲：

* 貼り付け方式切り替え
* 改行処理
* コピーのみ
* 右クリックコピー
* クリップボード復元
* 確認して貼り付け

## Phase 5：ボタンマネージャー

目的：
JSONを直接編集せず、GUIで管理できるようにする。

実装範囲：

* グループ一覧
* ボタン一覧
* 本文編集
* 追加・編集・削除
* 並び替え
* 複製
* 保存・キャンセル
* プレビュー

## Phase 6：高度機能

目的：
日常利用で便利な補助機能を追加する。

候補：

* `{clipboard}` 変数
* `{date}` 変数
* `{time}` 変数
* 選択テキスト取り込み
* ホットリロード
* インポート／エクスポート
* 自動バックアップ
* プロファイル切り替え
* 検索機能

## 15. 初期実装で優先すべきこと

最初に確認すべき技術的リスクは、以下。

1. **WPFウィンドウをクリックしてもフォーカスを奪わないこと**
2. **その状態でボタンクリックイベントが取れること**
3. **Webアプリの入力欄に `Ctrl + V` が届くこと**
4. **Windows Terminal / Claude CLI に貼り付けできること**
5. **半透明・Topmost・ドラッグ移動が問題なく共存すること**

ここが動けば、Prompletの核は成立します。

## 16. 現時点のMVP仕様

最初に作るべき最小版はこれです。

```text
Promplet MVP

- Windows / C# / WPF
- 小型パレット
- Topmost
- 半透明
- ドラッグ移動
- フォーカスを奪わない
- 固定ボタン3〜5個
- ボタンクリックで定型文をクリップボードに入れる
- Ctrl+Vで貼り付け
- まずはChatGPT、Claude、Windows Terminalで動作確認
```

このMVPが通れば、その後にJSON、常駐、マネージャーの順で拡張するのがよいです。
